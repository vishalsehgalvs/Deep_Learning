2 · Why RNN Instead of ANN?

Consider sentiment classification of reviews:

1. I love Sheryians          → 1  (positive)
2. Sheryians provide quality → 1  (positive)
3. I hate Sheryians          → 0  (negative)

One-Hot Encoding (OHE) — the ANN approach

Vocabulary size = 7 words. Each sentence → matrix of OHE vectors:

Sentence 1 → [ [1,0,0,0,0,0,0],   ← "I"
               [0,1,0,0,0,0,0],   ← "love"
               [0,0,1,0,0,0,0] ]  ← "Sheryians"
① Text input size may vary
② High computational power
③ Train/test input size mismatch
④ Semantic sequence NOT retained

5 · Backpropagation Through Time (BPTT)

Loss Function — Binary Cross-Entropy
L = -y·log(ŷ) - (1-y)·log(1-ŷ)
Weight Updates — Gradient Descent
Wᵢ_new = Wᵢ - η · (∂L/∂Wᵢ)
Wₕ_new = Wₕ - η · (∂L/∂Wₕ)
Wₒ_new = Wₒ - η · (∂L/∂Wₒ)

Chain rule via BPTT:
∂Loss/∂Wₒₗd = (∂Loss/∂O₃) × (∂O₃/∂O₂) × (∂O₂/∂O₁) × (∂O₁/∂Wₒₗd)

6 · Problems with Vanilla RNN

① Long-term memory is weak
Forgets early words in long sentences
② Vanishing Gradient
Gradient → 0; early layers stop learning
③ Exploding Gradient
Gradient → ∞; training becomes unstable
④ No long dependencies
Can't link "Rahul" to "he" 2000 words later
⑤ Sequential processing is slow
Can't be parallelized across tokens
Simple RNNs are rarely used today — LSTM and GRU were invented specifically to solve these problems.

7 · LSTM — Long Short-Term Memory

RNN Memory
Short-term only

Forgets context after a few tokens

LSTM Memory
Long-term Short-term

Maintains both via separate streams

LSTM State Variables

SymbolNameRepresents
Cₜ₋₁Previous cell stateLong-term memory
hₜ₋₁Previous hidden stateShort-term memory
xₜCurrent inputWord at this timestep
hₜCurrent hidden stateOutput / new short-term memory
CₜCurrent cell stateUpdated long-term memory

LSTM Architecture — 3 Gates

① Forget Gate — What to DISCARD from long-term memory
fₜ = σ( Wf · [hₜ₋₁, xₜ] + bf )   output ∈ [0,1] — 0=forget, 1=keep

Example — "Riya moved to London" (forget "Delhi", keep "gender", "profession"):

ContextGate valueAction
Gender0.99Keep
Profession0.95Keep
Delhi0.10Forget
Hospital0.85Keep
London0.90Keep (new)
② Input Gate — What NEW info to ADD to cell state
iₜ = σ( Wᵢ · [hₜ₋₁, xₜ] + bᵢ )      ← 2nd filter
C̃ₜ = tanh( Wc · [hₜ₋₁, xₜ] + bc )    ← candidate cell state

Cell state update:
Cₜ = fₜ * Cₜ₋₁ + iₜ * C̃ₜ            ← scale old + scale new
③ Output Gate — What to OUTPUT as hidden state
oₜ = σ( Wₒ · [hₜ₋₁, xₜ] + bₒ )
hₜ = oₜ * tanh(Cₜ)                   tanh squashes Cₜ to [-1,1]

8 · GRU — Gated Recurrent Unit

LSTM had 3 gates — complex, slow. GRU simplifies to 2 gates:

Problems with LSTM
① Complex architecture
② More training parameters
③ Higher time complexity
GRU Solution

Merges forget + input into a single update gate.
Merges cell state + hidden state.
Same performance, fewer parameters.

GRU Equations

Update Gate zₜ — Keep old memory or replace?
zₜ = σ( Wz · [hₜ₋₁, xₜ] )   ∈ [0,1]
Reset Gate rₜ — How much past influences new info?
rₜ = σ( Wr · [hₜ₋₁, xₜ] )   ∈ [0,1]
Candidate + Final Hidden State
h̃ₜ = tanh( W · [rₜ * hₜ₋₁, xₜ] )   ← candidate hidden state
hₜ  = (1 - zₜ) * hₜ₋₁ + zₜ * h̃ₜ    ← mix old + new

GRU Example — "Riya moved to London"

GRU summary: (1) creates new info from the word, (2) mixes it with old memory, (3) mixing ratio decided by update gate.

11 · All Formulas at a Glance

RNN
hₜ = tanh( xₜ·Wᵢ + hₜ₋₁·Wₕ + b )
ŷ  = σ( hₜ·Wₒ )
LSTM
fₜ = σ(Wf·[hₜ₋₁,xₜ] + bf)           ← Forget gate
iₜ = σ(Wᵢ·[hₜ₋₁,xₜ] + bᵢ)           ← Input gate
C̃ₜ = tanh(Wc·[hₜ₋₁,xₜ] + bc)        ← Candidate cell state
Cₜ = fₜ * Cₜ₋₁ + iₜ * C̃ₜ            ← Cell state update
oₜ = σ(Wₒ·[hₜ₋₁,xₜ] + bₒ)           ← Output gate
hₜ = oₜ * tanh(Cₜ)                   ← New hidden state
GRU
zₜ = σ(Wz·[hₜ₋₁,xₜ])                ← Update gate
rₜ = σ(Wr·[hₜ₋₁,xₜ])                ← Reset gate
h̃ₜ = tanh(W·[rₜ*hₜ₋₁, xₜ])         ← Candidate hidden state
hₜ = (1-zₜ)*hₜ₋₁ + zₜ*h̃ₜ            ← Final hidden state
Loss + Optimisation
L     = -y·log(ŷ) - (1-y)·log(1-ŷ)  ← Binary cross-entropy
W_new = W_old - η · (∂L/∂W)                ← Gradient descent
